# 抖音对标读取流程

## 当前能力

项目里已经补了一个本地抖音读取工具：

```text
tools/douyin_read.py
```

以及一个本地视频转写工具：

```text
tools/transcribe_video.py
```

它们当前能做 5 件事：

1. 解析抖音分享链接。
2. 读取视频标题、作者、点赞、评论、分享、收藏等基础信息。
3. 获取公开视频下载链接，并保存视频到本地。
4. 从视频中抽取若干画面帧，用于观察字幕、画面节奏和视觉结构。
5. 使用本地 faster-whisper 生成逐字稿、SRT 字幕和结构化 JSON。

- 视频 ID：7580689481808743686
- 点赞量：约 910 万
- 评论量：约 11 万
- 收藏量：约 298 万
- 分享量：约 143 万

视频和抽帧已保存到：

```text
douyin_samples/
```

本地转写结果已保存到：

```text
transcripts/
```

转写模型：

```text
faster-whisper small
```

转写结果可用于表达结构拆解，但会有少量同音错字，需要人工校对后再作为正式引用。

## 使用方式

### 只读取基础信息

```powershell
C:\Users\fun\pipx\venvs\douyin-mcp-server\Scripts\python.exe tools\douyin_read.py "抖音分享链接" --pretty
```

### 下载视频

```powershell
C:\Users\fun\pipx\venvs\douyin-mcp-server\Scripts\python.exe tools\douyin_read.py "抖音分享链接" --pretty --download-to douyin_samples
```

### 下载并抽帧

```powershell
C:\Users\fun\pipx\venvs\douyin-mcp-server\Scripts\python.exe tools\douyin_read.py "抖音分享链接" --pretty --frames-to douyin_samples --frame-count 8
```

### 尝试语音转写

```powershell
C:\Users\fun\pipx\venvs\douyin-mcp-server\Scripts\python.exe tools\douyin_read.py "抖音分享链接" --pretty --transcribe
```

语音转写需要额外配置：

```text
DASHSCOPE_API_KEY
```

未配置时，工具会跳过转写，不会影响基础解析、下载和抽帧。

### 使用本地 Whisper 转写视频

```powershell
C:\Users\fun\Documents\自媒体商业计划\.venv-whisper\Scripts\python.exe tools\transcribe_video.py "本地视频路径.mp4" --output-dir transcripts --model small --language zh --timestamps
```

输出 3 个文件：

```text
.txt   逐字稿
.srt   字幕文件
.json  结构化转写结果
```

## 当前限制

### 已经可用

- 单条抖音视频解析。
- 单条视频下载。
- 关键帧抽取。
- 通过字幕画面辅助分析表达方式。
- 本地 Whisper 逐字稿生成。

### 还不稳定或暂不可用

- 直接读取完整评论区。
- 像真人一样刷推荐流。
- 抖音全站搜索。
- 完全准确的自动逐字稿。

### 需要补充的能力

如果后续要稳定拆对标视频表达方式，最好补齐：

1. 人工校对逐字稿：修正 ASR 同音错字。
2. 评论抓取能力：用于判断观众为什么被打中。
3. 更密集抽帧 + OCR：用于对照画面字幕和音频转写。

## 对项目的用法

后续分析对标视频时，建议按这个顺序：

```text
抖音链接
→ 解析基础信息
→ 下载视频
→ 抽帧看字幕和视觉结构
→ 用本地 faster-whisper 转写完整口播
→ 拆开头、转折、论证、案例、收束、互动钩子
→ 转成我们人物拆解栏目的表达模板
```

## 这条张予不才视频的初步表达启发

仅从标题和抽帧看，它有几个明显优点：

1. 开头不是讲人物生平，而是直接改写大众认知。
2. 标题使用“不是因为 A，而是因为 B”的强转折结构。
3. 画面大量使用历史影像和纪录片质感素材，降低口播枯燥感。
4. 字幕短句清楚，适合知识内容快速跟读。
5. 右侧有关键词标注，帮助观众抓住本段主题。

这类结构可以迁移到高手人物拆解：

```text
宫崎骏之所以是宫崎骏，不是因为他想象力丰富，而是因为他从来不相信灵感会自己出现。
```

```text
乔布斯之所以厉害，不是因为他会做产品，而是因为他知道什么东西绝对不能出现在产品里。
```
