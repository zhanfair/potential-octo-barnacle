# 宫崎骏第一条视频结构稿二轮复审

## 这次复审的目的

这不是推翻上一轮修订，而是在升级后的模型视角下，再跑一遍同一个审稿流程，专门找：

- 上一轮没有发现的结构重复。
- 当前主线里还不够锋利的地方。
- 更适合进入逐字稿前的表达顺序。
- 哪些点应该现在改，哪些点应该等硬资料回来再改。

## 二轮总判断

当前稿件已经从“能讲”进入“有栏目范式”的状态。

新的判断是：

```text
这条视频真正要证明的，不是“宫崎骏很会创作”，而是“高手不是不可学习的天才，高手有一套处理模糊问题的动作系统”。
```

宫崎骏只是第一位样本。第一条视频如果成立，观众记住的应该不是“宫崎骏真厉害”，而是：

```text
原来高手可以被拆成动作。
```

这是比上一轮更重要的定位。

## 新发现 1：现在有一个“痛点重复”

上一轮把普通创作者痛点提前到第 1 段，这是对的。

但原稿第 3 段仍然保留了原来的“创作者真正的处境”：

```text
一个创作者最痛苦的地方是什么？
不是没有想法。
恰恰相反，是脑子里有太多感觉。
```

这会造成两个问题：

1. 前 90 秒里同一层意思出现两次。
2. 视频中段本来应该进入“问题如何被拆开”，却又回到了“你很痛苦”。

### 建议

第 1 段负责讲痛点：

```text
你不是没有想法，你是还没有把想法压成一个可以推进的动作。
```

第 3 段不要再讲痛点，而要改成“问题定义”：

```text
创作真正难的不是产生想法，而是给模糊感受找到一个可以被推进的界面。
```

这样后面接“分镜 = 思考界面”才顺。

## 新发现 2：三证据可以升级成三层系统

上一轮把三证据改成：

```text
分镜
手作
现场/生命经验
```

这是对的。但二轮看下来，它还可以再压成更强的系统表达：

```text
分镜 = 思考界面
手作 = 反馈链路
现场 = 生命记忆库
```

这三个不是并列材料，而是一套动作系统的三层：

| 层级 | 宫崎骏动作 | 对普通创作者的翻译 |
|---|---|---|
| 思考界面 | 分镜 | 把模糊想法放到一个可移动、可修改、可推进的界面里 |
| 反馈链路 | 手作 | 让材料立刻告诉你哪里不对，而不是在脑子里自我感动 |
| 生命记忆库 | 现场观察 | 让作品里的动作来自真实经验，而不是只来自风格模仿 |

### 建议

后续逐字稿里不要只说“三个证据”，而要说：

```text
宫崎骏不是靠灵感，他有一套处理模糊的系统：
先把感觉放进分镜这个界面，
再用手作建立最短反馈，
最后用现场经验给动作注入生命。
```

这句话可以成为整条视频的中轴。

## 新发现 3：开头还可以更像“刀口”，少一点解释

当前开头是：

```text
宫崎骏之所以是宫崎骏，不是因为他想象力丰富。
这句话听起来像在故意反常识。
但它真正要反的，不是宫崎骏的想象力……
```

这个开头是清楚的，但略微解释型。二轮判断：第一条视频的前 10 秒可以更狠一点。

### 更锋利的开头方向

```text
如果你把宫崎骏解释成天才，
那这条视频就已经没法讲了。

因为天才这个词，
最容易让普通人放弃思考。
```

然后再接：

```text
你一说他是天才，
就不用再问他到底做了什么；
也不用再问，一个普通创作者能不能借走其中一个动作。
```

这个版本比“不是因为想象力丰富”更像账号态度，也更能建立你的解释者身份。

## 新发现 4：普通创作者的“可借走动作”应该更具体

当前结尾已经说：

```text
写下一句话。
画出第一格。
列出三个场景。
先做一个丑版本。
```

这很好，但还可以更贴近你的账号目标受众：知识型内容创业者、个人 IP 起步者。

### 建议把可借走动作改成内容创作者版本

```text
当你还没想清楚的时候，不要继续在脑子里等。

先写一句反常识判断。
先列三个能证明它的场景。
先做一版很粗糙的结构稿。
先把它讲给一个真实的人听。

你不是靠想清楚才开始，
你是靠开始，才知道哪里没想清楚。
```

这比“画出第一格”更贴近你的观众。

## 新发现 5：AI 争议可以更晚、更轻、更有杀伤力

上一轮已经把 AI 争议从第三证据改成反向验证，这是对的。

二轮进一步判断：AI 争议不应该承担太多解释，它只需要打一个“反证钉子”。

### 建议压缩成一小段

```text
这也是为什么，我们不能把宫崎骏那段 AI 争议简单理解成“老人反技术”。

他真正受不了的，不是工具变快。
而是一个动作只有形状，却没有身体经验。

那不是效率问题。
那是作品里没有人。
```

这段不要展开太多，否则会把第一条视频带偏到 AI 站队。

## 二轮圆桌审稿

### 主编 / 账号守门人

- 判断：现在最大机会是让第一条视频承担“栏目宣言”的功能。
- 亮点：已经不是人物传记，而是高手动作拆解。
- 问题：稿子还没有明确告诉观众“以后这个账号都会这样拆高手”。
- 必改：在开头或结尾加一句轻量栏目承诺：`我们看高手，不是为了崇拜，而是为了拆出一个能借走的动作。`

### 安先生结构审稿人

- 判断：主线比上一版清楚，但中段仍有重复和层级问题。
- 亮点：三证据已经回到同一命题。
- 问题：第 1 段和第 3 段都在讲“想法很多但落不下来”。
- 必改：第 3 段改成“问题定义：模糊感受需要一个推进界面”。

### 舒老师文案审稿人

- 判断：金句已经有了，但还需要让句子更像人讲出来。
- 亮点：“天才这个词，最容易让普通人放弃思考”能记住。
- 问题：“生命经验”“动作系统”容易像概念词。
- 必改：所有概念词后面跟一个动作画面：走路、停顿、回头、铅笔、纸、改掉一格。

### 观众代表

- 判断：我愿意看，但我最关心的是“我能怎么用”。
- 亮点：宫崎骏这个人物有吸引力。
- 问题：如果我不是动画爱好者，分镜和手作可能离我有点远。
- 必改：每个宫崎骏动作后面都加一句内容创作者翻译。

### 反对者 / 冷水评委

- 判断：最危险的不是结构，而是证据回来以后撑不住。
- 亮点：稿子已经知道自己需要时间码。
- 问题：如果找不到“现场/生命经验”的硬证据，第三证据会变虚。
- 必改：资料补强时优先找第三证据；找不到就把第三证据降级为“AI 反向验证”，不要硬讲。

### 知识库承接官

- 判断：这条视频适合沉淀成三张方法卡。
- 亮点：动作系统很清楚。
- 问题：现在还没把三张卡命名好。
- 必改：沉淀为 `思考界面卡`、`反馈链路卡`、`生命经验卡`。

## 二轮争议点

### 争议 1：开头要用“宫崎骏不是靠想象力”，还是“天才让人停止思考”？

- 分歧：前者更像标题，后者更像账号态度。
- 为什么重要：第一条视频不只是讲宫崎骏，还要建立你这个账号的观察方式。
- 最终方向：口播开头用“天才让人停止思考”，标题仍可保留“不是因为想象力”。

### 争议 2：第三证据“现场/生命经验”是否过虚？

- 分歧：它在结构上很漂亮，但需要硬资料支撑。
- 为什么重要：这条视频最容易从高手拆解滑向主观抒情。
- 最终方向：保留，但设为资料补强优先级最高的一项。找不到硬证据就改弱，不硬撑。

### 争议 3：要不要现在回改主稿？

- 分歧：有些发现已经很明确，有些要等资料回来。
- 为什么重要：过早回改可能把结构稿改得太细，后面资料回来还要返工。
- 最终方向：现在只回改两个确定项：去掉痛点重复、补三层系统表达。其他等硬资料回来。

## 建议的二轮结构

```text
0. 刀口开头：如果你把宫崎骏解释成天才，这条视频就没法讲了
1. 账号承诺：看高手不是为了崇拜，而是为了拆一个能借走的动作
2. 普通创作者处境：你不是没有想法，你是没有把想法压成动作
3. 问题定义：模糊感受需要一个可推进的界面
4. 核心判断：宫崎骏有一套处理模糊的动作系统
5. 第一层：分镜 = 思考界面
6. 第二层：手作 = 反馈链路
7. 第三层：现场 = 生命记忆库
8. 反向验证：AI 争议说明他反感的是没有身体经验的动作空壳
9. 承认代价：动作可借，代价不必照搬
10. 普通人动作：先写判断、列场景、做粗稿、给真人看
11. 意象收束：把一团雾一格一格逼成作品
```

## 建议现在回改的两处

### 回改 1：第 3 段改成“问题定义”

把原来的：

```text
一个创作者最痛苦的地方是什么？
不是没有想法。
恰恰相反，是脑子里有太多感觉。
```

改成：

```text
所以创作真正难的，不是产生想法。

难的是给一个模糊的感受，找到一个可以推进的界面。

如果没有这个界面，想法永远只能待在脑子里。
它会变得越来越大，越来越重，但就是落不下来。

宫崎骏厉害的地方，是他没有把创作停在脑子里。
他把模糊感受放进分镜、手作和现场里，
让它们一点点被动作逼问出来。
```

### 回改 2：核心判断后加“三层系统”

在第 4 段后加入：

```text
如果把这套方法拆开，它其实有三层。

第一层，分镜，是他的思考界面。
第二层，手作，是他的反馈链路。
第三层，现场，是他的生命记忆库。

这三层合在一起，才构成了宫崎骏处理模糊问题的动作系统。
```

## 暂时不建议回改的内容

- 不建议现在把全稿扩写成逐字稿。
- 不建议现在大幅增加 AI 争议。
- 不建议现在继续加宫崎骏生平。
- 不建议现在做标题终稿。

原因很简单：

```text
下一步硬资料会决定很多句子能不能成立。
```

## 二轮结论

这一轮最大的新增发现是：

```text
第一条视频不是在证明宫崎骏，而是在证明你的栏目方法。
```

所以它的核心不是：

```text
宫崎骏为什么厉害？
```

而是：

```text
为什么“天才”这个解释没用？我们到底能不能从高手身上拆出一个普通人能借走的动作？
```

这会让第一条视频更像账号的开场声明，而不是一条单独的人物视频。
